# 54개 법인 엑셀 검증 & 온보딩 작업 지침

> 이 문서는 54개 법인의 엑셀 파일을 FDS에 등록하기 위한 **체계적 작업 프로세스**입니다.
> Claude Code에게 이 문서를 참조하라고 하면 바로 작업 가능합니다.

---

## 핵심 원칙

**한 법인당 3단계**: 검증 → 학습 → 확인. 이 사이클을 54번 반복한다.

---

## 1단계: 엑셀 구조 검증 (법인당 5분)

### 확인할 것
법인에서 올라온 엑셀 파일을 열어서 아래 5가지를 확인한다.

| # | 확인 항목 | 왜 중요한가 | 기록 방법 |
|---|----------|------------|----------|
| 1 | **헤더 행 위치** | 파서가 1~60행을 탐색함. 60행 이후면 못 찾음 | 몇 행인지 기록 |
| 2 | **컬럼명** | 파서 키워드와 일치해야 자동 매핑됨 | 실제 컬럼명 그대로 기록 |
| 3 | **시트 구성** | 멀티시트면 각 시트별로 양식이 다를 수 있음 | 시트 수 + 각 시트 용도 |
| 4 | **합계/소계 행** | SKIP_KEYWORDS로 걸러야 하는 패턴 | 합계 표기 방식 기록 |
| 5 | **날짜 형식** | 패턴 학습의 일평균/월평균에 필요 | 날짜 형식 기록 (YYYY-MM-DD 등) |

### 현재 파서가 인식하는 키워드 목록

```
[수취인/거래처] → payee로 매핑
  수취인, 거래처, 거래처명, 받는, 상대방, 입금처, 업체, 예금주, payee

[금액] → amount로 매핑
  금액, 지급액, 출금, 이체금액, 원화금액, 지급금액, amount

[날짜] → date로 매핑
  일자, 날짜, 거래일, date

[계좌번호] → account로 매핑
  계좌번호, 계좌, account

[은행] → bank로 매핑 (계좌와 합쳐짐)
  은행, bank

[적요/메모] → memo로 매핑
  적요, 내용, 비고, 거래내용, 계정, 비용, 계정과목, memo, remark

[헤더 탐지용 키워드] (3개 이상 매칭되면 헤더 행으로 인식)
  거래처, 수취인, 금액, 계좌, 적요, 계정, 은행, 날짜, 일자,
  이체, 거래처명, 원화금액, 계정과목, 지급금액

[건너뛰는 행] SKIP_KEYWORDS
  합계, 소계, 총계, 총 계, 총현금, *, **, 경비, 정산,
  이체, 예금, 현금, 계좌명, 합 계
```

### 판단 기준

| 상황 | 결과 | 조치 |
|------|------|------|
| 컬럼명이 키워드 목록에 있음 | **바로 통과** | 그대로 업로드 |
| 컬럼명이 비슷하지만 정확히 안 맞음 | **파서 수정 필요** | 키워드 추가 요청 |
| 헤더가 60행 이후에 있음 | **파서 수정 필요** | 탐색 범위 확장 요청 |
| 합계 행이 특이한 패턴 | **SKIP 키워드 추가** | 패턴 추가 요청 |
| 날짜 형식이 특이함 | **파서 수정 필요** | 날짜 포맷 추가 요청 |
| 양식이 완전히 다름 (피벗, 가로형 등) | **별도 처리 필요** | 케이스별 논의 |

## 2단계: 데이터 학습 업로드

검증 통과한 법인의 과거 데이터를 업로드한다.

1. FDS 웹 접속 → 거래 업로드
2. 법인 선택
3. 파일 유형: **과거 데이터 (패턴 학습)**
4. 과거 엑셀 파일 업로드 (최소 3개월치 권장)
5. "패턴 학습 완료 · 신규 N건 추가" 메시지 확인

### 학습 데이터 품질 기준
- **최소**: 거래처 + 금액 컬럼이 인식되어야 함
- **권장**: 거래처 + 금액 + 날짜 (일평균/월평균 패턴 분석 가능)
- **최적**: 거래처 + 금액 + 날짜 + 계좌 + 적요 (전체 감지 규칙 활용 가능)

---

## 3단계: 검증 확인

학습 후 법인 관리 페이지에서:
- 학습 데이터: "등록됨" 뱃지 확인
- 분석 결과 탭에서 테스트 분석 1회 실행
- 이상/주의/정상 분류가 합리적인지 확인

## Claude Code에게 작업 요청하는 방법

### 파서 키워드 추가가 필요할 때
```
"XX법인 엑셀에서 컬럼명이 '받을곳'인데 payee로 매핑이 안 돼.
parse_single_sheet의 payee 키워드에 '받을곳' 추가해줘"
```

### SKIP 키워드 추가가 필요할 때
```
"XX법인 엑셀에서 '부서합계'라는 행이 걸러지지 않아.
SKIP_KEYWORDS에 '부서합계' 추가해줘"
```

### 날짜 형식 추가가 필요할 때
```
"XX법인 날짜가 '2024년03월15일' 형식이야.
calc_pattern_stats의 날짜 파싱에 이 포맷 추가해줘"
```

### 한 번에 여러 법인 등록할 때
```
"아래 법인들 corps.json에 등록해줘:
1. (주)ABC - 담당: hong
2. (주)DEF - 담당: kim
..."
```

## 효율적 작업 순서 (권장)

### 배치 1: 양식이 비슷한 법인끼리 묶기
같은 ERP를 쓰는 법인들은 엑셀 양식이 동일할 가능성이 높다.
→ 한 법인 검증하면 같은 그룹은 파서 수정 없이 통과

### 배치 2: 쉬운 것부터
1. **1라운드** (양식 단순, 키워드 매칭됨): 바로 업로드
2. **2라운드** (키워드 약간 다름): 파서에 키워드 추가 후 업로드
3. **3라운드** (양식 특이): 케이스별 파서 보완 후 업로드

### 작업 흐름
```
법인 엑셀 파일 받기
  ↓
헤더/컬럼 확인 (1단계)
  ↓
├─ 파서 호환 → 바로 업로드 (2단계) → 검증 (3단계) → 완료
│
└─ 파서 미호환 → 어떤 키워드/패턴이 안 맞는지 기록
                → Claude Code에게 파서 수정 요청
                → 수정 후 업로드 → 검증 → 완료
```
