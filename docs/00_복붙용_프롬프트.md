## 프로젝트 개요
지주사 재무팀 6명이 54개 그룹사 법인의 자금 최종승인 전 이상거래를 감지하는 웹 시스템(FDS).
각 담당자가 ~10개 법인 담당. 법인에서 엑셀로 올라오는 지출결의서를 분석.

## 배포

- GitHub: https://github.com/emjh0602-star/fraud-detector
- Railway.app (Volume: /app/data, git push → 자동 재배포)

## 기술 스택
- Backend: Python Flask + openpyxl + pandas + numpy (app.py 단일 파일)
- Frontend: 순수 HTML/CSS/JS (templates/index.html 단일 파일)
- 데이터: JSON 파일 기반 (history.json, corps.json, users.json, results.json)
- 보안: werkzeug password hashing, threading.Lock 파일 보호, 세션 24h

## 핵심 기능
1. 팀원별 로그인 + 관리자/일반 권한 분리
2. 법인별 담당자 지정 및 접근 제한
3. 엑셀 스마트 파서: 법인마다 다른 양식을 자동 인식 (헤더 1~60행 탐색, 컬럼 키워드 매핑)
4. 과거 데이터 누적 학습 (payee+amount+date 조합으로 중복 제거)
5. 이상거래 감지 11개 규칙 (가중치 합산, 4점↑=이상, 2~3점=주의)
6. 분석 결과 서버 저장 (법인별 최근 30개), CSV 내보내기

## 감지 규칙
- 5억↑ (+3), 1억↑ (+2), 5천만↑ (+1)
- 법인 평균 대비 5배 초과 (+2), 상위5% 금액의 2배 초과 (+2)
- 신규 거래처 (+1), 신규+1천만↑ (+1)
- 동일 수취인 당일 3건↑ (+1), 동일 금액 3건↑ 반복 (+1)
- 긴급 처리 요청 (+1), 계좌번호 미기재+100만↑ (+1)

## 파서 키워드 (현재 인식 가능)
- payee: 수취인, 거래처, 거래처명, 받는, 상대방, 입금처, 업체, 예금주
- amount: 금액, 지급액, 출금, 이체금액, 원화금액, 지급금액
- date: 일자, 날짜, 거래일
- account: 계좌번호, 계좌
- bank: 은행
- memo: 적요, 내용, 비고, 거래내용, 계정, 비용, 계정과목
- SKIP: 합계, 소계, 총계, 총현금, 경비, 정산 등
