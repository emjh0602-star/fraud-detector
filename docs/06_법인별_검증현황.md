# 법인별 검증 현황

> 법인 검증할 때마다 이 표에 기록한다. Claude Code가 이 표를 보고 진행 상황을 파악한다.

## 상태 범례
- **O**: 파서 호환, 학습 완료
- **△**: 파서 수정 후 완료
- **X**: 미완료
- **-**: 미착수

## 검증 현황

| # | 법인명 | 헤더행 | 인식된 컬럼 | 시트수 | 날짜형식 | 파서호환 | 학습건수 | 상태 | 비고 |
|---|--------|--------|------------|--------|---------|---------|---------|------|------|
| 1 | 더편한샵 | - | 거래처,금액,계좌,적요 | 1 | - | O | - | O | 1차 테스트 완료 |
| 2 | 디엔코스메틱스 | - | 거래처명,원화금액,계좌번호 | 1 | - | O | - | O | 1차 테스트 완료 |
| 3 | (법인3) | - | - | - | - | - | - | O | 1차 테스트 완료 |
| 4 | | | | | | | | - | |
| 5 | | | | | | | | - | |

## 파서 수정 이력

| 날짜 | 법인명 | 수정 내용 | 추가된 키워드/패턴 |
|------|--------|----------|------------------|
| (예시) | XX법인 | payee 키워드 추가 | '받을곳' |

## 발견된 엑셀 양식 유형

| 유형 | 특징 | 해당 법인 | 파서 호환 |
|------|------|----------|----------|
| A (표준) | 거래처/금액/계좌/적요 컬럼, 헤더 1~10행 | 더편한샵, 디엔코스메틱스 | O |

> 같은 ERP를 쓰는 법인은 같은 유형일 가능성이 높다. 유형을 분류해두면 나머지 법인 작업이 빨라진다.
