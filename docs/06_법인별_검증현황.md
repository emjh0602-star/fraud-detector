# 법인별 검증 현황

> 법인 검증할 때마다 이 표에 기록한다. Claude Code가 이 표를 보고 진행 상황을 파악한다.

## 상태 범례
- **O**: 파서 호환, 학습 완료
- **△**: 파서 수정 후 완료
- **X**: 미완료
- **-**: 미착수

## 발견된 엑셀 양식 유형

| 유형 | 특징 | 해당 법인 | 파서 호환 |
|------|------|----------|----------|
| A (표준) | 거래처/금액/계좌/적요 컬럼, 헤더 1~10행 | 더편한샵, 디엔코스메틱스 | O |
| A' (대웅그룹 표준) | 업체명/계정/금액/내용/은행명/계좌번호 | 힐리언스, (페이지원/HCC/IDS 등 추정) | O |
| D (category 없음) | 거래처/적요/금액/비고, 계정 컬럼 없음 | 대웅개발, 대웅이엔지 | O (calss 없이 학습) |
| 헤더 2줄 병합 | 7·8행 병합 헤더 + 섹션형 날짜 | 대웅인베스트먼트 | X (스킵) |

> 같은 ERP를 쓰는 법인은 같은 유형일 가능성이 높다. 유형을 분류해두면 나머지 법인 작업이 빨라진다.
